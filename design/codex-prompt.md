# Codex Prompt

你的任务是根据本目录中的截图和 UI 规格，实现 HTML-PPT Editor 的第一版界面。

## 需要先阅读的文件

请按顺序阅读：

1. `screen-spec.md`
2. `components.md`
3. `design-tokens.json`
4. `interaction-spec.md`
5. `acceptance-checklist.md`
6. `screenshots/*.png`

## 产品定位

HTML-PPT Editor 是一个 Chrome 插件，用于把 AI 生成的单文件 HTML PPT 或 HTML 汇报页变成可视化可编辑文件。

第一版目标：

- 让用户可以直接修改 HTML PPT 中的文字和基础版式。
- 尽量保留原 HTML PPT 的翻页、动画、快捷键和演示能力。
- 导出前可以预览确认。
- 导出新的 HTML 文件。
- 所有核心处理默认在本地完成。

## 实现要求

1. 不要只按截图像素复刻，要按组件化结构实现。
2. 保持当前插件已有功能，不要为了 UI 改造破坏现有编辑、预览和导出能力。
3. 侧边工作区必须与画布并列，不允许悬浮覆盖页面。
4. 工具栏和属性面板放在同一侧边工作区。
5. 选中态为蓝色实线框。
6. 编辑态为橙色虚线框。
7. 预览模式必须隐藏编辑器 UI。
8. 导出确认文案必须全部使用中文。
9. 优先复用现有代码结构，避免大规模重构。
10. 如果现有架构不支持一次性完成，请分步骤实现，但不要引入额外产品功能。

## 明确不做

第一版不要引入：

- AI 辅助改稿
- 账号体系
- 云同步
- 图片替换
- PDF / PPTX / 图片导出
- 多选、组合、批量对齐、批量移动
- 完整图层面板
- 复杂吸附线
- 模板库
- 团队协作

## 主要界面

### 1. 插件弹窗

- 主按钮：打开本地 HTML PPT 文件。
- 次级入口：编辑当前页面、打开多页示例、打开长页示例、问题信息、帮助。
- 显示 file:// 权限提示。
- 显示本地处理提示。
- 使用暗色工具型界面。

### 2. 编辑器选中态

- 左侧为主画布。
- 右侧为固定侧边工作区。
- 选中元素显示蓝色实线框和缩放控制点。
- 属性面板显示页面、位置尺寸、文字、对齐、层级、列表、修改记录、问题信息。

### 3. 编辑器文字编辑态

- 当前文本元素显示橙色虚线框。
- 显示文本光标。
- 支持局部选中文字加粗和改色。
- 编辑态下不要触发原页面翻页快捷键。

### 4. 预览和导出确认态

- 预览模式隐藏工具栏、属性面板、选中框、控制点。
- 顶部保留“返回编辑”和“确认导出”。
- 导出确认弹窗显示文件名、当前修改数量、是否已预览确认。
- 主按钮为“下载 HTML”。
